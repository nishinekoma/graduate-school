  偽装広告やダークパターンが蔓延する Web 環境
    偽装広告
      定義（https://www.deceptive.design/types/disguised-ads
        ダークパターンの一種であり、実際のコンテンツと広告の境界を意図的に曖昧
        にすることで、ユーザーを混乱させる手法。

      例
        人気のソフトウェアダウンロードサイトであるSoftpediaというサイトがある。
        希望するソフトウェアのダウンロードボタンに酷似した、目立つ広告のダウンロードボタンに表示すること。

    欺瞞的なデザインパターン（ダークパターン　https://en.wikipedia.org/wiki/Dark_pattern）
      URL：https://www.deceptive.design/
      定義
        ウェブサイト上でサイト閲覧者の操作に対して意図しない行動をとらせるUI（ユーザインターフェース）
        のパターンの総称。このパターンを集めたサイトがある。
          欺瞞パターンの種類
            https://www.deceptive.design/types
    
      例
        EC サイトで「残りわずか」「あと数分で終了」などの急かし表示
        購入手続き中に追加オプションが自動オンになるような UI
        解約手続きが何ページも深いところにある

    蔓延しているか
      https://www.itmedia.co.jp/news/articles/2312/20/news106.html
      Yahoo!広告、半年間で3914件の広告主を非承認や停止に　消費者だます「ダークパターン」増加。

  ユーザーの認知負荷を軽減
    定義
      ダークパターンなどの目的とは異なるUIを減らしたりすることにより、
      必要な情報だけを表示するようにすること。

  従来の AdBlock 等の技術は URL や静的構造に基づくパターンマッチングに依存
    AdBlock 等の技術とは？
      Webブラウザの広告ブロック拡張機能のことを指す。
      AdBlockはグーグルの拡張機能のこと。
    
    静的構造に基づくパターンマッチングとは？
      ブラウザを構成するURLやDOM構造を解析して、広告やダークパターンに一致する文字列や属性を
      探し、リクエストや要素を遮断するフィルタリング技術のこと。

    静的パターン（Static Pattern）
      ページの状態やユーザー行動に関係なく
      読み込まれた時点（ロード中）のコード・URLだけで判断できるパターン

      静的構造で分かる事
        ・DOM ツリー
        ・属性
        ・URL 文字列

      静的構造ではわからないこと
        ・実行後の状態
        ・意味の変化
        ・意図の裏切り

    動的パターン（Dynamic Pattern）
      定義
        時間経過・ユーザー操作・文脈変化によって
        意味や役割が変わる UI 振る舞いのことを指す。

      詳細
        サーバが返すまで遷移先不明や
        日替わり地域・端末で変わるなど。
        遷移先がAPIサーバを通じて返される。
        元から決まっていなとか。

      動的でしかわからない事例
        ・遷移先がAPIサーバを通じて返される。
        ・ユーザー状態依存 UI
          初回訪問 → 記事を表示
          2回目以降 → 広告全画面表示
            依存情報
              Cookie
              LocalStorage
              ログイン状態
        ・

  ここでの自然言語処理（NLP）とは？
    自然言語処理（NLP）
      コンピュータが 人間の言葉（自然言語）を理解・解析・生成する技術です。
      機械学習を使用してコンピュータが人間の言語を理解する技術

      計算言語学（人間の言語ルール・ベースのモデリング）、統計モデリング、機械学習（ML）、
      ディープラーニングを組み合わせることで、コンピューターやデジタル・デバイスがテキストや音声を認識、理解、生成できるようにします。
    
    手法
      １．文章を単語や形態素といった「小さな単位」に分割する前処理。
          例
            「これはテストです。」 → ["これ", "は", "テスト", "です"]
        
      ２. 単語埋め込み（Word Embedding）
        単語をベクトル（数値ベクトル）として表現し、意味的な関係性を学習する方法。
          例
            king ≈ queen, man ≈ woman がベクトルの演算で意味的に近くなる

      ３．構文・意味解析（Parsing & Semantic Analysis）
          文の文法構造や意味を解析します。
            構文解析
            意味解析
            文脈依存関係解析

      ４．分類・ラベリング（Text Classification / Token Classification）
          文章や単語をカテゴリーに分類する技術。
            例
              ・ダークパターンの可能性が高い/低い
              ・インターフェイス説明文を意味カテゴリに分類

      ５．質問応答（Question Answering）
          文章から質問に対して答えを抜き出すモデル。
            例
              「この UI の説明文の要点は？」 のような意味抽出
      
      ６．言語生成（Natural Language Generation）
        意味理解だけでなく、反対に自然な文章を生成する技術。
        （チャットボット/要約など）
    学習方式
      ✔ 教師あり学習
        ・ラベル付きデータを使いモデルを訓練する方式
        ・分類/抽出などの精度が高い
        ・「これはダーク」「これは正常」とラベル付き
        ・精度は高い
        ・データ作成が大変

        例
          悪意ある UI テキスト vs 正常なテキストのラベルデータで学習する

      ✔ 教師なし学習
        ・ラベルがないデータを使い構造やパターンを学習する方式
        ・ラベルなしで構造を発見
        ・異常検知など
        ・解釈が難しい

      ✔ 自己教師あり（最近主流）
        ・大量テキストから自動で学習
        ・BERT系・LLM
        
        例
          クラスタリング、語義解析など

    自然言語処理を用いて心的パターンを抽出する
      例
        意味分散
          「登録内容を確認します」
          「次に進むことでサービスが有効化されます」
          特徴
            単語として「課金」「定期購入」は存在しない
            意味としては 課金確定
              ➡ 構文は安全、意味が危険

        Yes/Noの意味反転
          [はい、広告を受け取りたい]
          [いいえ、節約する気はありません]
          ユーザーの意図：
            「広告はいらない」
          UIが誘導する心理：
            「“節約しない人”になりたくない」
          ➡ 選択を歪めている

  文脈に応じて動的に変化する欺瞞的 UI の検知には限界があった
    概要
      文脈に応じて動的に変化する欺瞞的UIとは、ユーザーの操作・時間・状態に応じて表示内容や意味が変化し、
      表面的には正当だが、実際の行為を誤認させるUIである。

    根拠
      https://arxiv.org/abs/2412.09147
        ダークパターン研究における3つの主要な課題、すなわち分類の不一致と不完全性、検出ツールの限界、
        そして既存データセットの不十分な包括性について論じます
      
      https://arxiv.org/abs/2303.06782
        コンピュータービジョンと自然言語処理技術を用いて、アプリケーションの
        スクリーンショットから10個の固有のUIダークパターンの存在を示す一連の視覚的およびテキスト的
        な手がかりを認識し、それらの検出、分類、および位置特定を可能にする、新しい自動化アプローチであるAidUI(画像＋テキストでも判定)
        評価の結果、\AidUIはダークパターンインスタンスの検出において、全体の適合率0.66、再現率0.67、F1スコア0.65を達成


    なぜ、自然言語処理（NLP）を用いると精度があがあるのか。

        

    何故自然言語処理＋動的 UI 検知が難しいのか
      AidUIより、「同じ種類のダークパターンでも多様な見た目を持つため、単純なルールでは検出が困難」
        →　同じ種類での多様性が挙げられる。

        1. デザインの自由度が高い
          同じ意図（誘導・誤認）でも色・配置・文言・画像を変えるだけで見え方が大きく変わる
            → ルールベースでは対応しづらい  
        2. 文脈依存性が強い
          ある文言やボタンがダークパターンか否かは、
          前後の文脈やユーザー行動履歴
          によって変わる場合がある
            → 静的に解析できない
        3. 動的生成・遅延表示がある
          JavaScript で動的に DOM を変更する UI（表示タイミングで誘導内容やラベルが変わる）
          → 静的スナップショットでは情報が欠ける

  他関連する要素
    はい、NLP 以外の方法も重要で、多く併用されます

    たとえば：

    ✔ コンピュータビジョン（画像として UI を解釈）
    ✔ 行動ログ解析（クリック/スクロール履歴を解析）
    ✔ 統計/機械学習モデルによるパターン検出
    ✔ ルールベースのヒューリスティック解析

    NLP は テキストの意味理解に強いですが、
    UI 解析全体では テキスト以外の情報（視覚/レイアウト/行動） も考慮する必要があるため、よく複数技術が組み合わされます。

  クラウド型AIによる解析は、通信環境による遅延の不安定さやプライバシー流出が、即時性を要する UI 介入において大きな障壁
    
    クラウド型AI
      インターネットを介してユーザーのデバイスからデータをクラウドサーバに送信し、
      クラウド上で AI モデルが推論（解析）を行った後、結果をユーザーに返す仕組み

        ・Web アプリからテキストや UI 情報を送る
        ・クラウド上の AI が解析（例：大規模言語モデル・NLP・分類モデルなど）
        ・結果を返す
        ・返ってきた結果を UI に反映する

    WebNN は W3C の標準化プロセス（候補段階）にあり、Chromium 系ブラウザでは実装が進んでいますが、
    安定版でデフォルトで有効になるという段階ではありません。実装にはフラグ有効化や開発版の利用が必要な例が多いです
    。従って、WebNN 利用は新規性がありながらも実装成熟度に大きなばらつきがある技術領域であり、実装と評価という意味
    で修士研究としての価値があります。
        

    人間の反応時間（代表的な参考値（古典的 HCI 研究から）
      目で認識( ≈ 100ms)　　            大体0.1秒　
      判断を伴う反応時間（200〜500 ms）  大体0.2 ~ 0.5秒
      意思決定を伴う反応（500 ms〜2 s）　大体0.5 ~ 2秒



  ⑥「数百 ms でも十分では？」への最終回答（これを言えると強い）

    数百 ms の遅延が常に保証されるなら問題ありません。
    しかし UI 介入は、一度でも間に合わなければ失敗する性質を持ちます。
    そのため、平均遅延ではなく、最悪遅延をゼロに近づける設計が求められます。
    これを満たせるのが、完全クライアントサイド推論です。

    　→つまり、従来のクラウド型推論とローカル推論でどの程度ダークパターンが防げるのか
    　　などの評価が必要になってくると考えられます。

  ユーザーインターフェース（UI）
    定義
      工業デザイン分野におけるUIは「人間と機械のインタラクション（相互作用）が行われる空間」を指します。
      このインタラクションの目的は、人間側から機械を効果的に操作・制御すると同時に、機械側からオペレーター
      の意思決定を支援する情報をフィードバックすること
